# TutorMap Architecture

TutorMap은 지역 기반 SEO 정적 사이트 생성기입니다.

이 문서는 TutorMap의 페이지 생성 규칙과 허브 구조를 정의합니다.
앞으로 모든 생성기 구현은 이 문서를 기준으로 합니다.

## 핵심 원칙

TutorMap의 모든 페이지는 하나의 허브입니다.

모든 허브는 부모 허브를 가질 수 있습니다.
모든 허브는 자식 허브를 무한히 가질 수 있습니다.

페이지를 먼저 생성하지 않습니다.
항상 허브 구조를 먼저 정의하고, 그 허브를 기준으로 페이지를 생성합니다.

## 기본 사이트 구조

TutorMap의 기본 구조는 다음 순서를 따릅니다.

```text
전국과외
↓
시도과외
↓
시군구과외
↓
읍면동과외
↓
과목과외
↓
학년과외
↓
학교과외
```

이 기본 구조는 절대 변경하지 않습니다.

## 허브 구조

모든 키워드는 허브입니다.

예를 들어 `구로구과외`는 하나의 허브입니다.

`구로구과외` 아래에는 다음과 같은 하위 허브가 생성될 수 있습니다.

```text
구로구영어과외
구로구수학과외
구로구국어과외
구로구과학과외
```

`구로구영어과외`도 하나의 허브입니다.

`구로구영어과외` 아래에는 다음과 같은 하위 허브가 생성될 수 있습니다.

```text
구로구초등영어과외
구로구중등영어과외
구로구고등영어과외
```

`구로구고등영어과외`도 하나의 허브입니다.

`구로구고등영어과외` 아래에는 다음과 같은 학교 허브가 생성될 수 있습니다.

```text
구로고영어과외
신도림고영어과외
```

모든 페이지는 허브입니다.

## 확장성 원칙

새로운 키워드가 추가되어도 기존 구조를 변경하지 않습니다.

예를 들어 다음 키워드가 추가될 수 있습니다.

```text
성인영어
유아영어
토익
토플
SAT
방문과외
화상과외
```

새로운 키워드는 기존 허브 아래에 새로운 허브로 추가합니다.

기존 허브의 의미나 위치를 변경하지 않습니다.
기존 URL 구조를 변경하지 않습니다.

## 지역 데이터 원칙

지역 데이터는 엑셀에서만 관리합니다.

생성기는 엑셀 값을 그대로 사용합니다.
생성기는 엑셀에 없는 새로운 지역명을 만들지 않습니다.

지역 데이터의 기본 흐름은 다음과 같습니다.

```text
엑셀
↓
DataLoader
↓
Python 객체
↓
허브 Tree
↓
Route 생성
↓
SEO 생성
↓
HTML 생성
```

JSON, CSV, DB는 지역 데이터의 원본으로 사용하지 않습니다.

## URL 규칙

TutorMap은 한글 URL만 사용합니다.

예시는 다음과 같습니다.

```text
/전국과외/
/서울과외/
/강남구과외/
/대치동과외/
/대치동영어과외/
/대치동고등영어과외/
/휘문고영어과외/
```

영문 slug는 사용하지 않습니다.
지역명과 키워드는 한글 값을 유지합니다.

## SEO 원칙

모든 페이지는 허브 페이지입니다.

모든 허브 페이지는 가능한 경우 다음 허브들과 연결합니다.

```text
상위 허브
하위 허브
형제 허브
관련 허브
```

SEO 구조는 허브 관계를 기준으로 생성합니다.
페이지 간 내부 링크는 허브 Tree를 기준으로 생성합니다.

## 생성 원칙

지역은 데이터입니다.

페이지는 허브입니다.

생성기는 지역 데이터를 이용하여 허브를 만들고, 허브를 기준으로 페이지를 생성합니다.

생성 순서는 다음 원칙을 따릅니다.

```text
지역 데이터 로딩
↓
허브 Tree 생성
↓
Route 생성
↓
SEO 생성
↓
HTML 생성
```

페이지 생성은 항상 허브 Tree 이후에 수행합니다.

## 절대 변경 금지

TutorMap에서 다음 원칙은 변경하지 않습니다.

```text
Python 생성기 유지
정적 HTML 유지
한글 URL 유지
허브 구조 유지
엑셀 기반 유지
```

새로운 기술을 도입하기보다 기존 Python 기반 정적 사이트 생성기를 확장합니다.

## 구현 기준

앞으로 구현되는 Builder, DataLoader, TreeBuilder, RouteBuilder, SEOBuilder, Renderer는 이 문서의 규칙을 기준으로 동작해야 합니다.

코드는 이 문서의 허브 구조, 한글 URL, 엑셀 기반 데이터 원칙을 손상시키면 안 됩니다.
